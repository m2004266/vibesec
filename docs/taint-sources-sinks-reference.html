VibeSec Taint Sources and Sinks Reference

A practical documentation page for common taint-analysis sources and sinks across popular languages and frameworks. This page helps developers understand what to customize in VibeSec rules.

Important: This is a common reference, not a complete list of every possible source and sink in every framework. VibeSec should provide built-in presets and allow developers to add project-specific custom sources and sinks.

1. Simple Taint Analysis Model

Source

A source is where untrusted data enters the application.

request parametersrequest bodyheaderscookiesuploaded files

Flow

The value moves through variables, string concatenation, helper functions, or object properties.

req.query.id → id → sql

Sink

A sink is a sensitive function where untrusted data can become dangerous.

SQL executioncommand executionHTML outputfile access

Sanitizer / Safe Handling

A sanitizer or safe pattern reduces the risk before data reaches a sink.

parameterized queriesallowlist validationoutput encodingsafe parsers

4. VibeSec Policy Example

This is how a developer can enable default and taint rules in .vibesec.yaml.

presets:
  - vibesec:default
  - vibesec:taint

severity:
  minSeverity: warning

files:
  exclude:
    - "**/node_modules/**"
    - "**/vendor/**"
    - "**/*.test.*"

customTaint:
  sources:
    - "ctx.request.body"
    - "req.query.$FIELD"
  sinks:
    - "runSql(...)"
    - "executeRawQuery(...)"

5. Simple Semgrep Taint Rule Shape

VibeSec can convert documented sources and sinks into Semgrep-compatible taint rules.

rules:
  - id: vibesec.custom.sql-injection
    mode: taint
    languages: [javascript, typescript]
    severity: ERROR
    message: Untrusted input reaches a SQL execution sink.

    pattern-sources:
      - pattern: $REQ.query.$FIELD
      - pattern: $REQ.body.$FIELD

    pattern-sinks:
      - patterns:
          - pattern: $DB.query($QUERY, ...)
          - focus-metavariable: $QUERY

    pattern-sanitizers:
      - pattern: Number(...)
      - pattern: parseInt(...)

    metadata:
      cwe: "CWE-89"
      owasp: "A03:2021 Injection"
